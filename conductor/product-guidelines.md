# Product Guidelines

## 1. UX/UI Principles
- **Stitch Design Adherence:** The UI must faithfully implement the designs provided in the Google Stitch project. Any deviations require design review.
- **Desktop-First:** Optimize interactions for mouse and keyboard. Ensure robust keyboard navigation, shortcuts, and focus states.
- **Responsiveness:** Layouts must fluidly adapt to window resizing. Define minimum window sizes to prevent broken layouts.
- **Platform Conventions:** While maintaining brand consistency, respect fundamental OS paradigms (e.g., standard window controls on macOS vs. Windows).

## 2. Branding & Style
- **Visual Language:** Use the typography, color palette, and iconography defined in the Stitch design system.
- **Tone of Voice:** In-app copy should be clear, professional, and helpful. Avoid overly technical jargon when communicating with the user.

## 3. Quality & Performance
- **Snappy Interactions:** UI elements should respond immediately to user input. Use optimistic UI updates where appropriate.
- **Accessibility:** Ensure high contrast ratios, readable text sizes, and screen reader compatibility.
